1. Major Security / Performance Issues

- Missing Directory fsync (src/util/atomic_write.rs): The comment explicitly claims the centralized recipe includes "fsync of the parent dir" to protect against corruption, but the code only calls sync_all() on the temp file itself before persist() (rename). Without fsyncing the parent directory, the atomic rename isn't actually guaranteed to survive a crash or power loss.
- Heap Churn in Logging (src/daemon/supervisor.rs): The RingBuffer logs llama-server stdout/stderr by allocating and dropping a String continuously for every line once the 4096 capacity is reached. For a high-throughput process, this causes entirely unnecessary heap churn. It should just pop the oldest string, clear() it, and reuse the allocation to read the next line.

2. Dumb Things & Duplication

- The Phantom fd-lock Migration: In Cargo.toml, fd-lock = "4" was added along with a comment stating it "# replaces the hand-rolled PID lockfile (audit §2.1 #1)". However, src/daemon/lockfile.rs STILL uses the hand-rolled libc::flock implementation. The crate was added to the build tree, but the code was never actually migrated.
- tempfile Overhead (src/util/atomic_write.rs): tempfile was also added to replace hand-rolled atomic states. However, you're still manually calling std::fs::create_dir_all(dir) before invoking tempfile::Builder::new().tempfile_in(dir).

3. Replaceable by Existing Libraries (LoC Reduction)

- Hand-rolled GGUF Parsing (src/gguf/\*): You have over ~2,000 lines of code dedicated to hand-rolled GGUF parsing logic (header.rs, identity.rs, metadata.rs, memory.rs). The entire format logic could be deleted and replaced with the established gguf crate from the Rust ecosystem.
- Hand-rolled JSON-RPC & Framing (src/ipc/\*): src/ipc/framing.rs implements a custom length-prefixed protocol codec over tokio::io::duplex, and src/ipc/methods.rs implements a custom JSON-RPC router.
- Fix: tokio-util (with the codec feature) provides a battle-tested LengthDelimitedCodec out of the box. The RPC layer could be replaced by a minimal library like jsonrpc-core or tarpc to strip down the boilerplate.
